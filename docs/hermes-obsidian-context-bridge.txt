Hermes Obsidian context bridge
==============================

Hermes Console writes the current Obsidian attachment state to:

  <vault>/.obsidian/hermes/context.json

This bridge is a file, not a second Obsidian plugin. The architecture is:

  1. Obsidian plugin: Hermes Console captures selected text or cursor context
     from the active note and writes the bridge file.
  2. Bridge file: .obsidian/hermes/context.json stores the latest attach/null
     marker for the active vault.
  3. Hermes companion integration: hermes/obsidian_context_bridge.js runs inside
     Hermes, reads the bridge, injects context with pre_llm_call, and exposes
     obsidian_context() for large selections.

End-to-end context requires all three pieces. If the JSON file updates but
Hermes does not see the context, debug the Hermes companion loading/env wiring;
do not add another Obsidian plugin.

Plain Enter always writes a fresh marker before the PTY receives Enter. When
"Send Obsidian context to Hermes" is off, the marker has attach.enabled=false
and context=null. Hermes consumers must treat that as a fresh detach and clear
any previously accepted selected text.

Standalone Hermes loader
------------------------

This repository includes a concrete CommonJS loader at:

  hermes/obsidian_context_bridge.js

Load it from Hermes with the bridge path for the active vault:

  const { createObsidianHermesBridge } = require("/path/to/lean-obsidian-terminal/hermes/obsidian_context_bridge");

  const obsidian = createObsidianHermesBridge({
    bridgePath: "/Users/danny/wiki/.obsidian/hermes/context.json"
  });

  module.exports = {
    pre_llm_call(turn) {
      return obsidian.pre_llm_call(turn);
    },
    tools: {
      obsidian_context() {
        return obsidian.obsidian_context();
      }
    }
  };

If the Hermes runtime uses a different plugin shape, keep the same calls:

  1. Call obsidian.pre_llm_call(turn) immediately before the LLM request.
  2. Register obsidian.obsidian_context() as the obsidian_context tool.

pre_llm_call reads the latest bridge file, rejects stale payloads, accepts a
newer updateTimestamp even if submitSequence reset after an Obsidian/plugin
reload, and appends a system message to turn.messages when that array exists.
It also returns the injection string for runtimes that inject prompt material
themselves.

obsidian_context() returns the full selected text or cursor context for the
current accepted turn. It returns null after a fresh disabled/null marker, after
the freshness window, or when the bridge file is missing/invalid.

Environment variable fallback
-----------------------------

If bridgePath is omitted, the loader reads OBSIDIAN_CONTEXT_BRIDGE_PATH.

  OBSIDIAN_CONTEXT_BRIDGE_PATH="/Users/danny/wiki/.obsidian/hermes/context.json"

Expected behavior
-----------------

- Missing/unreadable bridge: pre_llm_call returns "" and obsidian_context()
  returns null.
- attach.enabled=false: pre_llm_call returns "" and obsidian_context() returns
  null, clearing previously accepted context.
- Selection context: pre_llm_call injects metadata plus selected text when small;
  large selections inject a preview and obsidian_context() returns the full text.
- Cursor context: pre_llm_call injects the file path, cursor, current line, and
  capped surrounding lines.
